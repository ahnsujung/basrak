## Project Overview

바스락 (Basrak) — citizen science forest fire risk monitoring platform. Citizens report leaf dryness (4 levels) and wind speed (6 levels) via mobile; data feeds a real-time national risk map. Connected to Korea's National Forest Science Institute.

Korean language is used throughout the UI and documentation.

## Commands

```bash
npm run dev       # Vite dev server at localhost:5173
npm run build     # Production build to /dist
npm run lint      # ESLint (flat config, v9)
npm run preview   # Preview production build locally
```

No test framework is configured. No Makefile.

## Tech Stack

- **Frontend**: React 19 + Vite 8 + Tailwind CSS 4.2
- **Backend**: Supabase (PostgreSQL + Auth + Storage + Realtime)
- **Maps**: Leaflet.js with Korea GeoJSON (not Kakao Maps — migrated)
- **PWA**: vite-plugin-pwa (autoUpdate, manifest in public/)
- **Deployment**: Vercel (auto-deploy from GitHub)
- **Edge Functions**: Deno (supabase/functions/)
- **Path alias**: `@/` resolves to `src/`

## Architecture

### Layer separation (strict)

```
pages/        → Route entry points — compose components + call hooks ONLY, no inline logic
components/   → UI units — can include logic, must NOT be page-specific
hooks/        → Data/state logic — NO UI code
lib/          → External service clients (Supabase init at lib/supabase.js — single instance)
utils/        → Pure functions (risk calculation, formatting)
```

### Key data flows

**Observation submission** (useObservation): photo upload → insert observation → insert point_log → update profile points → check milestones → update streak

**Map rendering** (useMapObservations): fetch 24h observations + subscribe to Supabase Realtime INSERT → render as circles (<100 obs) or heatmap (>=100 obs, IDW interpolation cached in daily_maps table)

**Streak** (useProfile): consecutive daily observations tracked in KST, with 1-day miss shield. Backfill logic from point_logs if last_observed_date is missing.

**Local contribution** (useLocalContribution): province detection via hardcoded bboxes, 10km radius stats, 7-day window.

### Database (Supabase PostgreSQL)

- **profiles** — extends auth.users (nickname, level, total_points, streak_days, last_observed_date, streak_shield)
- **observations** — lat/lng, dryness_level(1-4), wind_level(1-6), risk_score(generated: dryness+wind, range 2-10), photo_url
- **point_logs** — audit trail (reason: 'observation', 'milestone_*', 'streak', 'bonus')
- **daily_maps** — cached heatmap grid data (JSONB), generated by edge function

RLS is enabled on all tables. Trigger `on_auth_user_created` auto-creates profile on signup.

Migrations: `supabase/migrations/YYYYMMDD_NNN_description.sql`

### Environment variables

Accessed via `import.meta.env.VITE_*`:
```
VITE_SUPABASE_URL
VITE_SUPABASE_ANON_KEY
VITE_VWORLD_KEY          # optional, for VWorld map tiles
```

## Conventions

- Mobile-first layout: 430px max-width centered container (PageLayout)
- All dates/times use KST (UTC+9)
- Risk colors: green (#4CAF50) ≤3, yellow (#FFC107) ≤5, orange (#FF9800) ≤7, red (#F44336) >7
- npm uses `legacy-peer-deps=true` (.npmrc)
- Supabase client initialized once in `src/lib/supabase.js` — never re-initialize elsewhere
- Map component: `LeafletMap.jsx` (Leaflet.js 기반)
