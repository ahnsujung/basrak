### 1. 첫 화면 지도 공개 + 관찰 시 로그인 유도
- `/` 라우트에서 `RequireAuth` 제거 → 비로그인도 지도 바로 확인
- `/observe`, `/profile`은 `RequireAuth` 유지
- `RequireAuth`에서 `/auth`로 리디렉션 시 `state: { from: 현재경로 }` 전달
- `Auth.jsx`에서 로그인 완료 후 `from` 경로로 복귀 (없으면 `/`)
- 관찰 페이지는 메뉴·UI 열람 가능, **관찰 등록하기** 버튼 클릭 시에만 로그인 요구

### 2. 지도 타일 → VWorld white + 행정경계 GeoJSON
- CartoDB Positron(영문) → VWorld white 시도 → `gray` 레이어 없음 확인 → `white` 레이어 정상 동작
- 최종: VWorld 타일 완전 제거, **시도 행정경계 GeoJSON**으로 교체
  - 소스: `https://cdn.jsdelivr.net/gh/southkorea/southkorea-maps@master/kostat/2013/json/skorea_provinces_geo_simple.json`
  - 17개 시도, `name` 필드에 한국어 지명 포함
  - 스타일: 흰색 육지(`#f8fafc`), 연파란 바다 배경(`#dbe9f4`), 회색 경계선
  - 시도 이름 `bindTooltip(permanent: true)` + `.province-label` CSS
- **주의**: VWorld WMTS 유효 레이어는 `Base, midnight, Hybrid, Satellite, white`뿐 — `gray`는 없음

### 4. Session 8-1 — 스트릭 + 홈 뱃지 + 지역 기여 카운터

#### DB 마이그레이션 (Supabase에 적용 필요)
```sql
alter table profiles
  add column streak_days integer default 0,
  add column last_observed_date date,
  add column streak_shield integer default 1;
```

#### 스트릭 (연속 관찰 일수)
- `useObservation.js`에 `updateStreak(userId)` 추가
  - KST(`UTC+9`) 기준 날짜 비교
  - 오늘 이미 기록 → 변경 없음
  - 어제 기록 있음 → streak +1
  - 하루 이상 끊김 + shield 보유 → shield -1, streak +1 (복구)
  - shield 없으면 streak = 1 리셋
- `useProfile.js` 백필 로직: `last_observed_date`가 null이면 `point_logs`에서 KST 기준 연속일 자동 계산 후 DB 저장
- `Profile.jsx`: `profile.streak_days` 큰 숫자로 표시 (주황색 카드)

#### 홈 TopBar 등급 뱃지
- `LevelBadge`에 `size="sm"` prop 추가 → 원형 아이콘만 표시
- `Home.jsx` TopBar `right` 슬롯에 뱃지 배치, 클릭 시 `/profile` 이동

#### 스트릭 배너 (`StreakBanner.jsx`)
- TopBar 아래 "N일째 연속 관찰 중" 표시
- shield 보유 시 "🛡 복구권 보유" 표시
- `streak_days`가 0이면 렌더링 안 함

#### 지역 기여 카운터 (`useLocalContribution.js` + `LocalContributionCard.jsx`)
- 지도 영역 최상단(`top-3`)에 카드로 표시
- 내 주변 10km · 최근 7일 관찰자 수 + 내 기여율
- 17개 시도 경계 좌표 기반 누적 관측 수 집계 → 전국 순위
- 순위 표현 규칙:
  - 1~3등: `🥇/🥈/🥉 XX지역 전국 N등!`
  - 4~10등: `XX지역 전국 N등`
  - 11등 이상: `XX지역 N등`
- 위치 권한 없으면 자동으로 표시 안 됨

- Leaflet + Tailwind CSS 4: `h-full` = 0px → `absolute inset-0` 사용
