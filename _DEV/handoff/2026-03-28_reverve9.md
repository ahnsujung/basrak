# Handoff - 2026-03-28 — 구조 보완 + 테마 시스템 계획

## 현재 상태 진단

코드베이스 전체 점검 완료. 기능은 동작하지만 아래 구조적 문제가 누적되어 있음.

## Phase 1: 구조적 결함 수정 (우선)

### 1-1. 에러 핸들링 전면 추가 ⚠️ P0

**문제**: Supabase 호출 대부분에 `.catch()`가 없음. 네트워크 에러 시 UI가 영구 로딩 상태로 멈춤.

**수정 대상**:
| 파일 | 현재 | 수정 |
|------|------|------|
| `hooks/useLocalContribution.js` | Promise.all → catch 없음 | .catch + error 상태 추가 |
| `hooks/useMapObservations.js` | Promise.all → catch 없음 | .catch + error 상태 추가 |
| `hooks/useProfile.js` | .then만 있음 | .catch 추가 |
| `components/profile/MyObservations.jsx` | .then만 있음 | .catch 추가 |
| `components/profile/PointHistory.jsx` | .then만 있음 | .catch 추가 |

참고: `useObservation.js`만 유일하게 try/catch가 있어서 패턴 참고 가능.

---

### 1-2. 컴포넌트 내 직접 fetch → 훅 분리 ⚠️ P1

**문제**: 프로젝트 원칙 "hooks/ → 데이터, components/ → UI만" 위반.

**수정**:
- `components/profile/MyObservations.jsx`의 supabase 호출 → `hooks/useMyObservations.js` 신규
- `components/profile/PointHistory.jsx`의 supabase 호출 → `hooks/usePointHistory.js` 신규
- `components/map/ObservationLayer.jsx`의 `renderHeatmap()` 내 supabase 호출 → 훅 또는 상위에서 prop 주입

---

### 1-3. App 에러 바운더리 추가 ⚠️ P1

**문제**: 에러 바운더리 없음. Leaflet 지도 초기화 실패 시 전체 앱 크래시.

**수정**:
- `components/ui/ErrorBoundary.jsx` 생성 (React class component)
- `App.jsx`에서 `<Routes>` 감싸기

---

### 1-4. 비동기 cleanup ⚠️ P2

**문제**: 비동기 작업 중 컴포넌트 언마운트 시 setState 호출 → 메모리 릭 경고.

**수정**: 각 훅의 useEffect에 mounted flag 패턴 적용:
```js
useEffect(() => {
  let mounted = true
  fetchData().then(data => {
    if (mounted) setData(data)
  })
  return () => { mounted = false }
}, [deps])
```

대상: `useLocalContribution`, `useMapObservations`, `useProfile`, 신규 훅 2개
